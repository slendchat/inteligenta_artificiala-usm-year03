Лабораторная работа 6 — Генетический алгоритм для f(x) на [0,1]

Кратко
- Представление решения в алгоритме: вещественное x ∈ [0,1]
- Для соответствия заданию в логах дополнительно показывается 4-битное кодирование x, но само решение остаётся вещественным как в исходной программе
- Поиск максимума f(x), x ∈ [0,1]
- Операторы: отсечение верхней половины (truncate selection), арифметический кроссовер, мутация малым шумом
- Подробные логи на английском человеческим стилем
- График изменения максимального fitness по поколениям

Запуск
- Базовый запуск с параметрами по умолчанию:
  `python lab06/genetic_alg.py`
- Настройка параметров:
  `python lab06/genetic_alg.py --N 8 --generations 40 --pc 0.9 --pm 0.05 --func v1 --seed 42`

Параметры
- `--N` — размер популяции, 2..20 (по умолчанию 4)
- `--generations` — число поколений G, минимум 30 (по умолчанию 30)
- `--pc` — вероятность кроссовера (по умолчанию 0.8)
- `--pm` — вероятность мутации отдельного бита (по умолчанию 0.05)
- `--func` — выбор целевой функции: `v1`, `v2`, `v3` (по умолчанию `v1`)
- `--seed` — фиксированное зерно для воспроизводимости (опционально)
- `--plot` — путь для сохранения графика максимального fitness (по умолчанию `lab06/max_fitness_by_generation.png`)

Целевая функция
- f(x) = x * sin(10 * x)

Представление решений
- Внутри алгоритма используется исходная логика: вещественное x ∈ [0,1]
- В логах дополняю представление в 4 бита: b = round(x * 15), bits = bin(b) при L = 4. Это только для отображения и не влияет на ход алгоритма

Логирование — что выводится
- Номер поколения
- Популяция в двоичном и десятичном виде (4 бита), x и fitness каждого индивида
- Сводные метрики fitness: max, min, avg
- Селекция — пояснение про отсечение верхней половины по fitness
- Кроссовер — для каждой пары родителей выводятся значения x родителей и alpha (арифметический кроссовер). Точки кроссовера не используются, это отражено в сообщении
- Мутация — сообщения о факте мутации и как изменился x конкретного потомка
- Итоговое x* и f(x*) и 4-битное представление лучшего решения

Графики и артефакты
- `lab06/max_fitness_by_generation.png` — график максимального значения fitness по поколениям
  Если matplotlib недоступен, программа корректно пропускает построение и сообщает об этом в лог

Эксперименты
- Проведите минимум три прогона, варьируя `--pc` и `--pm`. Например:
  1) `--pc 0.9 --pm 0.01`
  2) `--pc 0.8 --pm 0.05`
  3) `--pc 0.6 --pm 0.1`
- Сопоставьте скорость роста max fitness и стабильность результатов

Ответы на контрольные вопросы — кратко
- Хромосома — битовая строка длины L. Ген — отдельный бит, кодирующий часть значения переменной
- Fitness — значение целевой функции f(x) для декодированного x ∈ [0,1]
- Селекция — отбор родителей по приспособленности. Кроссовер — обмен подстроками для комбинирования признаков. Мутация — случайный flip битов для поддержания разнообразия
- Параметры `pc` и `pm` управляют обменом генетической информацией и степенью случайности, влияя на скорость и качество сходимости
- Поколение — полный цикл от оценки популяции, отбора, кроссовера и мутации до формирования новой популяции
- Разнообразие важно для избегания преждевременной сходимости и локальных максимумов
- Критерий останова — по числу поколений, по стагнации метрик, по достижению порога fitness

Замечания
- Внутренняя логика алгоритма не менялась, как в оригинальном файле. 4-битное представление добавлено только для соответствия формату вывода из задания
- Если потребуется другая целевая функция, её можно заменить в `math_func`
